# TravelVLM v3.1 — Modernisation UI/UX & Restructuration

**Objectif** : Refactoriser le codebase pour meilleure maintenabilité, performances et expérience utilisateur moderne.

**Durée estimée** : 3-4 heures (phased approach)

---

## Phase 1 : CSS Modernisation (30-45 min)

### État Actuel
- ✅ `style/style.css` existe avec variables CSS modernes
- ⚠️ 95% des pages utilisent **inline styles** (répétition, difficile à maintenir)
- ⚠️ Styles disparates : gradients, shadows, spacing inline dans chaque page

### Plan de Refactorisation

1. **Étendre `style/style.css` avec utilitaires et composants**
   - Créer classes réutilisables : `.header`, `.footer`, `.container`, `.card`, `.btn-*`
   - Ajouter utilitaires : `.mt-*`, `.p-*`, `.text-center`, `.flex`, `.grid`
   - Définir composants : `.hero`, `.section`, `.testimonial`, `.gallery-item`

2. **Remplacer inline styles progressivement**
   - Commencer par layouts critiques (header, footer, hero)
   - Utiliser find-replace pour patterns courants (`display: flex`, `margin`, etc.)
   - Tester chaque page après refactorisation

3. **Ajouter design tokens**
   - Espacement cohérent (8px grid)
   - Typographie standardisée
   - Palette de couleurs unifiée

---

## Phase 2 : Component Extraction (45-60 min)

### Objectif
Créer des composants réutilisables pour header, footer, navigation, hero, etc.

### Approche
1. **Créer `/components/` avec JavaScript partials**
   ```javascript
   // components/header.js
   function createHeader(branding = 'TravelVLM') {
     return `<header class="header">...</header>`;
   }
   ```

2. **Ajouter injection dans pages template**
   - Single `<script src="components/header.js"></script>`
   - Initialiser dans `DOMContentLoaded`

3. **Benefits**
   - DRY (Don't Repeat Yourself)
   - Maintenance centralisée
   - Cohérence visuelle globale
